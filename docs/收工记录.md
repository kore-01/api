## 一、本次完成的工作

### 1. MiniMax M2.5-highspeed 模型支持

**问题**: 调用 MiniMax API 时返回 404 错误

**解决**:
- 确认正确的模型名称：`MiniMax-M2.5-highspeed`（带前缀）
- 添加了两种 API 格式支持：
  - OpenAI 兼容格式 (`/v1/chat/completions`)
  - Anthropic 兼容格式 (`/anthropic/v1/messages`)
- 修复了 Anthropic 格式的转换逻辑：
  - 添加 `anthropic-version: 2023-06-01` 头部
  - `max_tokens` → `max_completion_tokens`
  - 移除不支持的 `tools` 参数

### 2. API Key 管理功能增强

**新增功能**:
- ✅ 创建 API Key 时可自定义 key 值
- ✅ 编辑现有 API Key（修改 key_value 或 strategy_id）
- ✅ 切换策略无需修改前端配置

**使用场景**:
> 前端配置了固定密码（如 `sk-4453974644643322270`）绑定了策略A，现在需要换成策略B，只需在后台修改 key 关联的策略即可，无需修改前端配置。

**相关文件**:
- `src/routes/apikey.routes.ts` - 后端 API
- `frontend/src/views/SettingsView.vue` - 前端界面

### 3. Provider 故障快速恢复

**问题**: Provider 故障后需要等待健康检查间隔（60秒）才能恢复

**解决**: 添加"立即恢复"按钮，可立即重置 Provider 状态

**位置**: 设置页面 → API Provider → 恢复按钮

### 4. 代码版本管理

**完成事项**:
- 初始化 Git 仓库
- 推送到 GitHub: `https://github.com/kore-01/api`
- 配置服务器从 GitHub 自动更新
- 创建一键更新脚本: `/opt/akdn/update.sh`

## 五、经验总结

### 遇到的问题

1. **404 错误**: 原因有两个
   - Anthropic 端点应该是 `/v1/messages` 而不是 `/chat/completions`
   - 模型名称必须是完整的 `MiniMax-M2.5-highspeed`

2. **400 错误**: Anthropic API 不支持 `tools` 参数，需要在转发前移除

3. **服务器编译文件未更新**: 本地修改后需要重新编译并部署到服务器

4. **Git 合并冲突**: 远程仓库有旧代码，使用 `--ours` 策略保留最新版本
